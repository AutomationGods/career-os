# Career OS North Star

The north-star product experience is a daily career command center that knows the user's verified career profile, understands current opportunities, tracks active pursuits, and recommends the highest-leverage next action.

Career OS should eventually answer every day:

- who the user is professionally based on verified facts
- what roles the user should pursue
- which jobs are worth attention
- which jobs should be ignored
- which applications are ready
- which claims need evidence
- which recruiters need follow-up
- which interviews need preparation
- what the highest-leverage next action is

The north star is not "generate a resume."

The north star is "guide the user through the right career pursuit process with truth, evidence, and momentum."

## Daily Career Mission

A mature Daily Mission should include:

- Best 5 opportunities to pursue
- 3 applications to complete
- 2 follow-ups due
- 1 recruiter relationship to warm up
- 1 interview prep task
- blocked claims needing evidence
- application packets ready
- jobs to ignore
- highest-leverage next action

The Daily Mission should be grounded in current state projections, not static checklists.

## Inputs to the north star

Daily guidance should be based on:

- career goal
- Profile Facts
- evidence gaps
- career profile summary
- job discovery results
- job intelligence classifications
- fit scores
- application packet statuses
- relationship history
- communication drafts and approvals
- interview events
- follow-up queue
- outcomes

## What the user should trust

The user should be able to trust that Career OS:

- does not invent experience
- does not claim unsupported credentials
- does not hide blocked claims
- does not send email without approval
- does not submit applications without approval
- does not scrape LinkedIn or bypass CAPTCHA
- shows why a job is recommended or ignored
- records important actions through events and projections

## What the UI should feel like

The UI should feel like an operating dashboard, not a document editor.

It should show:

- current professional profile
- current pursuit pipeline
- strongest opportunities
- active packets
- blocked or needs-evidence claims
- relationship tasks
- interview tasks
- follow-up tasks
- next action

Resume, cover letter, and message drafts should appear as artifacts inside a pursuit, not as the central product surface.

## What future agents should optimize for

Future agents should optimize for:

- truth source quality
- opportunity selection quality
- packet readiness
- relationship momentum
- interview readiness
- follow-up consistency
- user trust
- auditability

They should not optimize for application volume, fabricated positioning, or uncontrolled external actions.
